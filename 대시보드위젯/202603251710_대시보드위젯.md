## 2. chart-main 카드 레이아웃 확정

```
┌──────────────────────────────┐
│ [드래그핸들] 타이틀    [W×H] [✕] │  ← widget__header
├──────────────────────────────┤
│        ● 28.3 °C             │  ← cm-value-area (flex:0 auto, 중앙정렬)
│         +0.1 ▲               │    signal dot + 값 + 단위 + delta/trend
│  환기설정/난방설정 24.5 / 12  │    subRow(row2) 보조지표
├──────────────────────────────┤
│  ╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌ 28.3°C   │  ← cm-spark-area (flex:1, 남은공간)
│    ∿∿∿∿∿∿∿∿∿∿∿∿∿∿●          │    SparklineSVG (currentColor)
│ 28.1°C ╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌    │    min/max 점선 + 값 레이블
└──────────────────────────────┘
```

### 2-1. 크기 정책 (반응형)

| gridSize | 표시 내용 |
|----------|----------|
| w≥3, h≥2 | 값 + 스파크라인 (기본) |
| w<3 또는 h<2 | 값만, 수직 중앙 (`chart-main--compact`) |

### 2-2. 기본 그리드 크기
- 기본: `w:4 h:3` → 20컬럼 기준 한 줄 5개
- 신규 추가 시 `chart-main` → `w:4 h:3` / 기타 → `w:5 h:5`

## 3. SparklineSVG 확정 사양

- **알고리즘**: Catmull-Rom → Cubic Bezier 변환 (tension=0.3)
- **ViewBox**: `200 × 60`, PAD=6
- **Flat 데이터 처리**: range=0이면 y=H/2 중앙 수평선, area fill 없음
- **min/max 기준선**: range>0일 때만 점선 표시 (`strokeDasharray="4 3"`)
- **min/max 레이블**: range>0일 때만 실제 데이터 값 표시
  - max: `top: 10%`, `right: 14px`
  - min: `bottom: 12%`, `left: 14px`
- **색상**: `currentColor` / `var(--accent)` — 임의 색상 prop 없음

## 4. 색상 정책 (현재)

> 색상 정책 도입 전 임시 규칙

| 요소 | 색상 | 변경 방법 |
|------|------|----------|
| 스파크라인 선/점 | `var(--accent)` via `currentColor` | `.cm-spark-area { color: ... }` 한 줄 |
| 스파크라인 그라데이션 fill | `var(--accent)` 22%→0% | SparklineSVG 내 gradient stop |
| 신호등 dot | 임계값 없음: `#3f607a` / 경고: `#fb923c` / 위험: `#f87171` | `kpiStatusResolver` 연동 |
| 카드 좌측 보더 컬러 | **제거** (단일 `var(--border)` 사용) | - |
| 배경 그라데이션 | 상태 기반 (아래 참조) | Widget 클래스 |

### 4-1. 임계값 배경 그라데이션

| 상태 | 클래스 | 배경 |
|------|--------|------|
| 정상 | (없음) | `var(--surface)` |
| STALE_WARN | `widget--warn` | `linear-gradient(150deg, var(--surface) 25%, rgba(251,146,60,0.18) 100%)` |
| STALE_CRIT | `widget--crit` | `linear-gradient(150deg, var(--surface) 25%, rgba(239,68,68,0.28) 100%)` |

> 백엔드 가이드라인 API(`warnMin/warnMax/critMin/critMax`) 연동 후 `resolveKpiStatus`가 동일 상태값 반환 → 별도 코드 수정 없이 적용
