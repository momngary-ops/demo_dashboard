# 대시보드 위젯 카드 타입 확정 — 202603251430

## 개요
`chart-main` 외 3가지 위젯 카드 타입을 추가한다.

## 신규 위젯 타입

| 타입 | 설명 | 기본 크기 |
|---|---|---|
| `chart` | Recharts 멀티라인 시계열 차트 (주 KPI + 오버레이) | 8×4 |
| `gauge-set` | 구동기 개도(창/커튼/3Way밸브) 수평 바 목록 | 5×4 |
| `status-panel` | 자수동 + 작동상태 패널 | 5×4 |

## 핵심 정책
- 신호 없는 항목(`zoneCapabilities.available`에 없는 ID)은 위젯에 표시하지 않음
- `allAvailableIds: Set<string>` → DashboardPage에서 useMemo로 계산 → Widget prop으로 전달

### 1. `src/constants/actuatorCandidates.js` (신규)

구동기 그룹 정의.

#### `GAUGE_SET_GROUPS`
| 그룹 ID | 타이틀 | 아이템 수 |
|---|---|---|

#### `STATUS_PANEL_GROUPS`
| 그룹 ID | 타이틀 | 비고 |
|---|---|---|
| `window` | 창 자수동 | runId 없음 (auto만) |
| `curtain` | 커튼 자수동 | runId 없음 |
| `device` | 장치 작동 | CO₂/보광등/훈증기/보일러/순환펌프 — runId 있음 |
| `aux` | 보조기기 | xass1~6 — runId 있음 |
| `heatcool` | 냉난방기 | XheatandCool1~5 — runId 있음 |

### 4. `src/pages/DashboardPage.jsx` (수정)

추가 import:
```js
import { GAUGE_SET_GROUPS, STATUS_PANEL_GROUPS } from '../constants/actuatorCandidates'
```

추가 useMemo/useCallback:

```js
// 전체 연결 ID Set
const allAvailableIds = useMemo(
  () => new Set(Object.values(zoneCapabilities).flatMap(z => z.available ?? [])),
  [zoneCapabilities]
)

// 구동기 폴링 — gauge-set / status-panel 위젯 ID만 수집
const actuatorSlotConfigs = useMemo(() => { ... }, [widgets])
const actuatorKpiSlots = useKpiPolling(actuatorSlotConfigs, firstZoneId, refreshKey)
const actuatorSlotMap = Object.fromEntries(actuatorKpiSlots.map(s => [s.id, s]))

// chart 오버레이 ID도 secondarySlotConfigs에 포함 (기존 subRows + overlayIds)

// widget config 변경 저장
const handleConfigChange = useCallback((widgetId, partialConfig) => {
  setWidgets(prev => {
    const updated = { ...prev, [widgetId]: { ...prev[widgetId], ...partialConfig } }
    saveToStorage(STORAGE_KEY_WIDGETS, updated)
    return updated
  })
}, [])
```

handleAddWidget sizeMap:
```js
const sizeMap = { 'chart-main': [4, 3], 'chart': [8, 4], 'gauge-set': [5, 4], 'status-panel': [5, 4] }
```

Widget에 추가 props:
```jsx
actuatorSlots={actuatorSlotMap}
allAvailableIds={allAvailableIds}
allCandidates={allCandidates}
onConfigChange={(partial) => handleConfigChange(i, partial)}
```

## 검증 시나리오

1. `npm run dev` — 빌드 오류 없음
2. WidgetPicker → "구동기 개도" → 창/커튼 그룹 클릭 → gauge-set 위젯 추가됨
3. WidgetPicker → "작동 상태" → 장치 그룹 클릭 → status-panel 위젯 추가됨
4. WidgetPicker → "차트" → KPI 선택 → LineChart 위젯 표시
5. 차트 위젯 "+" 버튼 → 오버레이 KPI 추가 → 멀티라인 표시
6. 미연결 항목은 위젯 내에 표시되지 않음 확인
