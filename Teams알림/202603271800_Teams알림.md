**정상 복귀 분기:**
```
복귀 감지 시
  ├─ wasInDelay (outSince != null)
  │    → flapCount++
  │    └─ count >= 3 → FLAPPING 알림 발송 + flapCount 리셋
  └─ wasSent(prev) + isInCooldown

clearCooldown(prev)  ← 항상 실행
```

**FLAPPING 발송 시 payload:**
```js
{ ...slot, dataStatus: 'FLAPPING', flapCount: count }
```

**RECOVERED 발송 시 payload:**
```js
{ ...slot, prevStatus: prev }
```

**경고 발송 시:** `flapCount` 리셋 (딜레이 경과 후 알림 성공)

### 2. `src/utils/teamsNotifier.js`

**MessageCard → Adaptive Card 전환**

페이로드 구조:
```json
{
  "type": "message",
  "attachments": [{
    "contentType": "application/vnd.microsoft.card.adaptive",
    "content": { ...AdaptiveCard... }
  }]
}
```

**카드별 스타일:**
| 타입 | 제목 | Container style |
|---|---|---|
| OUT_OF_RANGE | ⚠️ 임계치 이탈 | `warning` |
| FLAPPING | ⚠️ 반복 이탈 감지 | `warning` |
| STALE_CRIT | 🔴 데이터 수신 중단 | `attention` |
| SENSOR_FAULT | 🚨 센서 오류 | `attention` |
| SENSOR_LOST | 🔌 센서 연결 끊김 | `attention` |
| RECOVERED | ✅ 정상 복귀 | `good` |

**공통 레이아웃 구성요소:**
- `headerContainer(title, farmName, style)` — 컬러 스트립 + 제목/농장명
- `kpiRow(slot)` — 아이콘 + KPI명 + 구역
- `dataRow(...)` — 2열 데이터 (라벨/값/보조)

**경고 카드 하단 행:**
- OUT_OF_RANGE: 이탈 확정(`alertDelayMin분 이상 지속`) / 발생시각
- FLAPPING: 이탈 확정(`딜레이 내 N회 반복`) / 발생시각
- STALE_CRIT 등: 마지막 수신값 / 발생시각
- RECOVERED: 복구 시각 (단일 텍스트)

**export:**
```js
export async function sendTeamsAlert(webhookUrl, slot)     // OUT_OF_RANGE, FLAPPING, 장애
export async function sendTeamsRecovery(webhookUrl, slot)  // RECOVERED
```

## 알림 정책 최종 요약

```
이탈 감지 → 딜레이 타이머 시작
  ├─ 딜레이 내 복귀 → flapCount++
  │     flapCount < 3  → 타이머 리셋, 알림 없음
  │     flapCount >= 3 → FLAPPING 발송, flapCount 리셋
  └─ 딜레이 경과 유지
        쿨다운 중    → 알림 없음
        쿨다운 아님  → 경고 발송, cd키 기록, flapCount 리셋

정상 복귀
  ├─ cd키 있음 + 쿨다운 내 → RECOVERED 발송, cd키 삭제
  └─ cd키 없음 or 쿨다운 아님 → 알림 없음, cd키 삭제
```

## 미결 사항

- FLAP_THRESHOLD(3회) — 향후 가이드라인 설정 UI에서 조정 가능하도록 고려
- RECOVERED 인앱 알림 표시 여부 노이즈 우려 시 Teams 전송만 유지 가능
