## 개요

대시보드 KPI 슬롯에서 이상 상태 발생 시 Microsoft Teams 채널에 자동 알림을 보내는 시스템.
브라우저 CORS 제한으로 인해 FastAPI 서버(`mock_data_server.py`)를 프록시로 사용한다.

## 알림 발송 흐름

```
DashboardPage
  └─ useAlertNotifier(widgetKpiSlots)   ← 60초 폴링 사이클마다 감시
       └─ 상태 변화 감지 (prev ≠ curr)
            └─ sendTeamsAlert(webhookUrl, slot)
                 └─ POST /api/admin/notify/teams
                      └─ FastAPI → Teams Incoming Webhook
```

## 트리거 조건

| dataStatus | 의미 | 알림 여부 |
|---|---|---|
| `OUT_OF_RANGE` | yMin/yMax 임계치 이탈 | ✅ |
| `STALE_CRIT` | 데이터 10분 이상 미수신 | ✅ |
| `SENSOR_FAULT` | 센서 하드웨어 오류 | ✅ |
| `SENSOR_LOST` | 센서 capabilities 제거 | ✅ |
| `NULL_DATA` | 데이터 없음 | ❌ (노이즈 많아 제외) |
| `STALE_WARN` | 5분 경과 주의 | ❌ |
| `OK` | 정상 | ❌ |

## 쿨다운 로직

- **기본값**: 30분
- 같은 KPI + 같은 상태는 쿨다운 내 재발송 없음
- 정상(OK)으로 복귀 시 쿨다운 리셋 → 재이상 발생 즉시 재발송
- 저장 위치: `localStorage.getItem('alert:cd:{id}:{status}')`

---

## 설정 방법

1. Teams 채널 → `...` → **커넥터** → **Incoming Webhook** → URL 발급
2. 대시보드 **농장 등록/설정** 페이지 → **알림 설정** 섹션
3. Webhook URL 입력 → 활성화 토글 ON → **테스트 전송** 확인

### 알림 설정 localStorage 키
```json
// 키: "alert:config"
{
  "enabled": true,
  "webhookUrl": "https://xxxx.webhook.office.com/webhookb2/...",
  "cooldownMin": 30
}
```
