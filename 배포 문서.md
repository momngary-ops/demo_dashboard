# 플랫폼 mock-up 대시보드 — 배포 문서

> 작성일: 2026-03-20
> 프로젝트: React + Vite SPA
> Remote: https://github.com/momngary-ops/demo_dashboard.git

---

## 배포 방식

**서버 git clone + 빌드 (방식 B)**

Ubuntu 서버에서 직접 코드를 받아 빌드 후 Nginx로 서빙.
GitHub Actions 자동 배포로 확장 가능, 백엔드 추가 시 Nginx proxy_pass만 추가하면 됨.

---

## 브랜치 전략

- `main` = 프로덕션 브랜치
- 작업: `feature/*` 또는 `dev` → `main` merge 후 배포
- 향후: `main` push → GitHub Actions → 서버 자동 배포

---

## Step 1. 로컬 — feature → main merge + push

```bash
cd "C:\Users\박찬형\Desktop\플랫폼 mock-up\dashboard"
git checkout main
git merge feature/data-input-integration
git push origin main
```

---

## Step 2. Ubuntu 서버 — 초기 환경 설치

```bash
# Node.js 설치 (nvm)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install --lts

# Nginx 설치
sudo apt update && sudo apt install -y nginx

# 배포 폴더 생성
sudo mkdir -p /var/www/dashboard
sudo chown $USER:$USER /var/www/dashboard
```

---

## Step 3. Ubuntu 서버 — 클론 + 빌드

```bash
cd /var/www/dashboard
git clone https://github.com/momngary-ops/demo_dashboard.git .
npm install
npm run build
# → /var/www/dashboard/dist/ 생성됨
```

---

## Step 4. Nginx 설정

파일 생성: `/etc/nginx/sites-available/dashboard`

```nginx
server {
    listen 80;
    server_name _;   # 도메인 확정 시 교체

    root /var/www/dashboard/dist;
    index index.html;

    # SPA 라우팅 (모든 경로 → index.html fallback)
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 향후 백엔드 추가 시 이 블록만 추가
    # location /api/ {
    #     proxy_pass http://127.0.0.1:8000;
    # }

    gzip on;
    gzip_types text/plain text/css application/javascript application/json image/svg+xml;
}
```

```bash
sudo ln -s /etc/nginx/sites-available/dashboard /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

---

## Step 5. 업데이트 스크립트 (deploy.sh)

파일 위치: `/var/www/dashboard/deploy.sh`

```bash
#!/bin/bash
cd /var/www/dashboard
git pull origin main
npm install --production
npm run build
sudo systemctl reload nginx
echo "Deploy complete"
```

```bash
chmod +x /var/www/dashboard/deploy.sh
```

이후 업데이트 배포 시: `./deploy.sh` 한 줄로 완료.

## 향후 고도화 경로

```
현재: 수동 deploy.sh 실행
   ↓
다음: GitHub Actions → SSH → deploy.sh (push to main 시 자동)
   ↓
이후: 백엔드(Python) 추가 → Nginx /api proxy_pass + systemd 서비스
```

---

## vite.config.js 확인 사항

- `base` 설정 없음 → 기본값 `/` 유지 (배포에 적합)
- `server.proxy` 설정은 개발 서버 전용 — 빌드 결과에 영향 없음

---

## 용어 설명

| 용어 | 설명 |
|---|---|
| **빌드(build)** | 개발용 코드를 브라우저가 빠르게 읽을 수 있는 형태로 변환. `npm run build` 실행 |
| **Nginx** | 웹 서버 프로그램. 브라우저 요청을 받아 우리 파일을 보내줌 |
| **dist/** | 빌드 결과물 폴더. 실제 서버에서 서비스되는 파일들 |
| **deploy.sh** | 최신 코드 받기 + 빌드 + 서버 재시작을 한 번에 실행하는 스크립트 |
| **SPA 라우팅** | URL이 바뀌어도 항상 index.html로 돌아오게 하는 Nginx 설정 |
