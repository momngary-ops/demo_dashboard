※ 생육 데이터의 특성
   - 측정 주기: 일 단위 (환경·경영의 1분 단위와 다름)
   - 당일 내 값 고정(flat): 대부분의 항목이 하루에 한 번 갱신
   - 실시간 꺾은선 위젯보다 일별 트렌드 차트(바 or 꺾은선)가 더 적합
   - 단, 073·076(속도 지표)은 기후 조건 기반으로 분 단위 미세 변동 있음

================================================================================

[ 068 ] 초장 | chojang | cm
  판정: ✅ 일별 트렌드 꺾은선 위젯 최우선 추천
  의견: 줄기 선단까지의 높이. 재배 기간 내 지속 생장 (기준 22cm → 최대 35cm).
        일 단위 약 0.1~0.3cm 증가. 당일 내에는 값 고정.
        실시간 스파크라인 불필요. 대신 최근 14~30일 일별 꺾은선이 생육 속도 파악에 핵심.
        생장 정체(기울기 감소) 시 재배 환경 이상 신호로 활용 가능.
        현재값 KPI 카드 + 최근 14일 꺾은선 트렌드 병행 권장.

[ 069 ] 줄기굵기 | julggi | mm
  판정: ✅ 현재값 KPI 카드 + 일별 트렌드 (068과 쌍 구성 권장)
  의견: 주줄기 직경 (범위: 10~15mm). 영양 상태·생육 균형의 간접 지표.
        너무 굵으면 영양 과잉(과번무), 너무 가늘면 영양 부족 또는 도장.
        068(초장)과 함께 '생육 기본 지표' 세트로 구성.
        당일 내 고정값 → 분 단위 꺾은선 불필요.

[ 070 ] 엽수 | num_leaves | 장
  판정: ✅ 현재값 카드 + 적엽 이벤트 감지 위젯
  의견: 활성 잎 수. 3일에 1장씩 증가 → 14장 초과 시 적엽 후 4~6장으로 리셋.
        적엽(De-leafing) 시점에 값이 급감하는 독특한 패턴.
        일별 꺾은선에서 급감 구간 = 적엽 작업 이벤트로 활용 가능.
        현재값 카드로 표시하고 적엽 필요 구간(예: 12장 이상) 경보 표시 권장.

[ 071 ] 화방높이 | height_flower | cm (0.5 단위)
  판정: ✅ 현재값 카드 (일 단위 트렌드 참조)
  의견: 첫 화방까지의 높이. 0.5cm 단위 측정.
        지속 증가하는 성장 지표. 생육 단계 추적에 유용.
        068(초장)과 높이 비교 시 화방 착생 위치 비율 파악 가능.
        일별 꺾은선으로 화방 발달 속도 확인.

[ 072 ] 화방수 | num_flower | 개 (0.33 소수 표기)
  판정: ✅ 현재값 카드 (소수 표기 전용 UI 고려)
  의견: 발생한 화방(꽃송이) 수. 0.33 단계 소수 증가 규칙(0 → 0.33 → 0.66 → 0.99 → 1.0).
        소수 표기는 화방 전개 중간 단계를 나타내는 관행적 표기.
        ※ 위젯 표시 시 "5.33군" 형태로 명시적 표기 필요 (일반 사용자 혼동 가능).
        074(착과군)·077(수확군)·078(착색군)과 함께 '단계별 과방 현황' 세트 구성 강력 권장.

[ 073 ] 화방 전개속도 | speed_flower | 개/주
  판정: ✅ 꺾은선 위젯 추천 (유일하게 분 단위 실시간 변동)
  의견: 주간 온도·일사량 기반으로 분 단위 미세 변동하는 속도 지표.
        기본값 0.85개/주. 기후 조건 좋을 때 최대 +8% 상승.
        환경데이터와 연동: 내부온도(020)·내부일사량(008)이 높을수록 값 상승.
        076(착과속도)과 쌍으로 묶어 '생육 속도 지표' 차트 구성 권장.
        실시간 꺾은선이 의미 있는 유일한 생육 항목.

[ 074 ] 착과군 | fruit_loaded | 군 (0.33 소수 표기)
  판정: ✅ 현재값 카드 (072 화방수와 세트)
  의견: 실제 열매가 달린 군 수. 072(화방수)와 동일한 0.33 소수 증가 규칙.
        화방수(072)보다 착과군이 낮으면 착과율 저하 신호 가능.
        072와 함께 단계별 과방 현황 세트에 포함.

[ 075 ] 착과수 | num_fruit | 개
  판정: ✅ 현재값 KPI 카드 + 일별 트렌드
  의견: 현재 나무에 달려있는 열매 총 수.
        2일마다 1~2개 증가 + 7일 주기 수확 시 4~8개 감소하는 주기적 증감 패턴.
        수확 주기(약 7일) 전후 증감을 일별 꺾은선에서 명확히 확인 가능.
        목표 착과수 대비 현재 달성 여부 표시 권장.

[ 076 ] 착과속도 | speed_fruit | 개/주
  판정: ✅ 꺾은선 위젯 (073과 쌍 — '생육 속도' 멀티라인 구성)
  의견: 열매 맺히는 속도. 기본값 0.80개/주.
        073(화방 전개속도)과 동일하게 기후 조건 기반 분 단위 변동.
        073과 함께 '생육 속도 지표' 멀티라인 차트로 구성.
        예: 화방 전개속도(실선) vs 착과속도(점선) 비교.

[ 077 ] 수확군 | harvest_grp | 군
  판정: ✅ 현재값 카드 (078 착색군과 세트)
  의견: 수확이 완료된 군 번호. 7일마다 +1군 증가.
        078(착색군)과 1군 차이가 정상 패턴(착색군 = 수확군 + 1).
        두 값의 차이가 2 이상이면 착색 지연 또는 수확 지연 이상 신호.
        072·074와 함께 '단계별 과방 현황' 세트 완성.

[ 078 ] 착색군 | coloring_grp | 군
  판정: ✅ 현재값 카드 (077과 쌍 — '단계별 과방 현황' 세트 완성)
  의견: 현재 착색(숙성) 중인 군 번호. 정상: 수확군(077)보다 1군 높음.
        077과의 차이 모니터링이 핵심.
        차이 = 1: 정상 / 차이 > 1: 착색 과잉 → 수확 가속 필요 경보.
        072·074·077과 함께 '단계별 과방 현황' 세트 구성.

    068 초장, 069 줄기굵기, 070 엽수, 071 화방높이, 075 착과수

  ※ 생육 데이터 전용 고려사항:
     1. 일별 갱신 특성상 실시간 스파크라인(20포인트) 대신
        일별 데이터 20포인트(최근 20일)로 대체 설계 권장
     2. 소수 표기 항목(072 화방수, 074 착과군):
        "5.33군" 표기 방식에 대한 UI 가이드라인 필요
     3. 환경데이터 연동 고려:
        073·076 속도 지표 → 020(내부온도)·008(내부일사량) 연계 표시 가능
     4. 이벤트 표시 필요:
        070 엽수 급감 = 적엽 이벤트 / 075 착과수 급감 = 수확 이벤트
        일별 차트에 이벤트 마커(▼) 오버레이 권장
