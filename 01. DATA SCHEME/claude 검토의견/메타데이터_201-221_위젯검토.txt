        멀티팜 지원 시 농장 선택 드롭다운의 키값으로 활용.

[ 202 ] 사업자 등록 번호 | business_reg_number | 문자열

[ 203 ] 농장명 | farm_name | 문자열

        대시보드 모든 화면의 헤더 또는 TopBanner에 고정 표시 필수.
        멀티팜 지원 시 현재 선택된 농장 컨텍스트 표시 기준.

[ 204 ] 주소 | location_address | 문자열

  의견: "충남 태안군 원북면 발전로" 등 실제 위치.
        지도 API 연동 시 농장 위치 핀 표시에 활용.
        독립 위젯 불필요. 농장 정보 카드 또는 설정 화면에 표시.

[ 205 ] 관리자 연락처 | manager_contact | 문자열
  판정: ❌ 위젯 불필요 — 관리자 설정 화면 항목
  의견: 비상 연락처. 개인정보 처리 주의.
        대시보드 긴급 알림 발신 대상 설정에 활용 가능.

[ 206 ] 총 면적 (ha) | total_area_ha | ha
  판정: ⚠️ 농장 기본 정보 카드 표시
  의견: 3.0ha. 정적 값.
        면적 대비 수확량·전력비 등 단위 면적 KPI 계산 기준값으로 활용.
        예: 단위 면적 수확량 = 107(수확량) / total_area_ha
        독립 위젯 불필요. 농장 정보 카드에 포함.

[ 207 ] 총 면적 (평) | total_area_pyeong | 평

  의견: 7,300평. 206(ha)의 국내 관행 표기 중복.
        206만 유지하고 UI에서 단위 전환(ha ↔ 평) 토글로 표시 권장.

[ 208 ] 총 구역 수 | total_zones | 개

        구역 지도 UI에서 총 구역 레이아웃 렌더링 기준값.
        독립 수치 위젯 불필요. 구역 선택 UI의 기반 데이터.

[ 209 ] 재배 작물 | crop_type | 문자열
  판정: ✅ TopBanner / 헤더 상시 표시 권장
  의견: "완숙토마토" 등 주요 재배 작물.

        작물에 따라 대시보드 단위·임계값·위젯 구성이 달라질 수 있음.
        ※ 플랫폼 다작물 확장 시 작물 유형별 UI 프리셋 전환 기준값으로 활용.

[ 210 ] 품종 | crop_variety | 문자열

  의견: "데프니스" 등 품종명.

        독립 위젯 불필요.

[ 211 ] 정식일 | planting_date | 날짜

        현재 날짜 기준 재배 경과일 자동 계산에 활용.
        예: "정식 후 D+199" 형태로 생육 단계 파악.
        생육 데이터(068~078)의 맥락 정보로 함께 표시 강력 권장.

        217과 함께 설비 기반 위젯 조건부 렌더링 패턴 적용.

[ 219 ] 보광 시스템 | lighting_system | 문자열 (Y/N)

종합 요약

  TopBanner 상시 표시 필수
    203 농장명, 209 재배 작물

  파생 카운터 카드 (계산 기반)
    211 정식일 → 재배 경과일 "D+N" 자동 계산
    212 수확 종료 목표일 → 재배 기간 진행률 게이지

  위젯 조건부 렌더링 제어 (핵심 설계 항목)
    217 냉방 → 044·045 위젯 활성화 기준
    218 CO₂ → 022·023·030·035 위젯 활성화 기준
    219 보광 → 031·036 위젯 활성화 기준
    ※ 이 세 값은 농장별 설비 구성에 따라 대시보드 위젯 세트를 자동 조정하는
       핵심 설계 패턴. 플랫폼 확장성에 매우 중요.

  시스템 연결 현황 카드
    220 센서 수 + 221 카메라 수 → 등록 대비 현재 연결 상태 모니터링

  구역 UI 기반 데이터
    208 총 구역 수 → 탭 개수 / zone_id·zone_name → 탭 레이블

  농장 정보 카드 (설정/정보 화면)
    204 주소, 206 면적, 210 품종, 211~212 재배 기간, 213~216 설비 구성

  위젯 불필요 (설정/관리 화면 전용)
    201 farm_id, 202 사업자번호, 205 연락처
    207 면적(평, 206 중복), 213~215 설비명
    zone_level remark

  ※ 메타데이터는 '위젯' 개념보다 '대시보드 초기화 설정값' 개념으로 접근 권장.
     실시간 데이터 API와는 별도로, 대시보드 최초 로드 시 한 번만 호출하여
     컨텍스트 및 조건부 렌더링 기준으로 활용하는 것이 적합.
