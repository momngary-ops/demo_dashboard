[ 051 ] 현재EC | now_ec | dS/m
  판정: ✅ 꺾은선 위젯 (050 set_ec과 멀티라인 쌍) / TopBanner KPI 카드 이미 구현
  의견: 실제 급액 EC. KPI 카드 및 mock 데이터 구현 완료.
        050(설정EC)과 멀티라인 쌍으로 구성 시 제어 추적 성능 시각화.
        EC 변동 패턴(관수 직후 희석 → 회복)을 꺾은선으로 잘 표현.

[ 052 ] 설정PH | set_ph | 1~14
  판정: ⚠️ 단독 위젯 불가 — 053(now_ph)과 멀티라인 쌍 필수
  의견: pH 목표 설정값. 050(설정EC)+051(now_ec) 쌍과 완전히 동일한 구조.
        053(현재PH)과 반드시 쌍으로 표시.

[ 053 ] 현재PH | now_ph | 1~14
  판정: ✅ 꺾은선 위젯 (052 set_ph와 멀티라인 쌍) / TopBanner KPI 카드 이미 구현
  의견: 실제 급액 pH. KPI 카드 구현 완료.
        pH 관리 범위(예: 5.8~6.2)를 경보 밴드로 표시 시 이탈 즉시 인지 가능.
        052와 멀티라인 쌍 구성.

[ 054 ] 일출시간 | sunrise | -
  판정: ❌ 위젯 불필요
  의견: 시각 문자열(예: 06:30). 독립 위젯 의미 없음.
        039(주야간상태) 차트 배경 음영, TopBanner 날씨 배경 전환 기준값으로 활용.

[ 055 ] 일몰시간 | sunset | -
  판정: ❌ 위젯 불필요
  의견: 054와 동일. 주야간 전환 기준값으로 백엔드 활용.

[ 056 ] 현재일사(W) | sunvol | W/m²
  판정: ✅ 현재값 위젯 (양액기 측 센서 — 006 xsunvol과 비교 검토 필요)
  의견: 양액기 시스템 측 일사 센서 현재값.
        006(xsunvol 복합환경제어기 측)과 같은 물리량일 수 있음.
        ※ 두 시스템이 동일 센서를 공유하는지, 별도 센서인지 확인 필요.
           - 동일 센서: 한 쪽만 사용, 중복 제거
           - 별도 센서: 두 값 비교로 센서 이상 감지에 활용 가능
        현재값 또는 꺾은선 위젯 적합.

[ 057 ] 누적일사(J/cm²) | sunadd | J/cm²
  판정: ✅ 게이지/진행바 위젯 권장 (관수 트리거 핵심 기준값)
  의견: 양액기 기준 누적일사량. 관수 자동 트리거의 실질적 기준값.
        007(xsunadd), 009(xinsunadd)과 유사하나 양액기 자체 계산값.
        목표 누적일사량 대비 현재 달성률을 게이지로 표시 시 직관적.
        관수 이벤트 발생 직후 값 리셋 여부도 확인 필요.

[ 058 ] 함수율(%) | water_con | %
  판정: ✅ 꺾은선 위젯 + 경보 임계값 / TopBanner KPI 카드 이미 구현
  의견: 배지 수분 함량. KPI 카드 및 mock 데이터 구현 완료.
        관수 후 급상승 → 배수 후 서서히 감소하는 톱니 패턴.
        하한 임계값(예: 45%) 이하 시 관수 지연 경보 표시 권장.
        꺾은선 위젯으로 가장 명확히 표현되는 데이터 중 하나.

[ 059 ] 배지EC(dS) | medium_ec | dS/m
  판정: ✅ 꺾은선 위젯 / TopBanner KPI 카드 이미 구현
  의견: 배지(코이어·암면 등) 내 EC. 뿌리 주변 염류 농도.
        급액EC(051)보다 높으면 배지 내 염류 집적 → 뿌리 스트레스.
        051(급액EC)과 함께 멀티라인으로 표시 시 비교 분석 용이.
        KPI 카드 구현 완료.

[ 060 ] 배지온도(℃) | medium_temp | ℃
  판정: ✅ 꺾은선 위젯 / TopBanner KPI 카드 이미 구현
  의견: 배지 내 온도 = 실질적 근권 온도.
        020(내부온도)과 함께 표시 시 공기온도-근권온도 차이 파악 가능.
        겨울철 저온 뿌리 스트레스 감지에 중요.
        KPI 카드 구현 완료.

[ 061 ] 실내온도(℃) | in_temp | ℃
  판정: ⚠️ 020(xintemp1)과 중복 가능성 높음 — 센서 출처 확인 필요
  의견: 양액기 시스템 측 실내온도 센서값.
        020(xintemp1 복합환경제어기 측)과 동일 센서 공유 여부 확인 필요.
        - 동일 센서: 중복, 한 쪽만 사용
        - 별도 센서 (양액기 설치 위치): 미세 차이가 근권 환경 분석에 유용
        ※ mock CSV에서 실제로 xintemp1과 in_temp 값이 거의 동일함 → 공유 센서 추정.
        독립 위젯 우선순위 낮음.

[ 062 ] 실내습도(%) | in_hum | %
  판정: ⚠️ 021(xinhum1)과 중복 가능성 높음 — 센서 출처 확인 필요
  의견: 061과 동일 논리. mock CSV에서 xinhum1과 in_hum 값이 동일 → 공유 센서 추정.
        독립 위젯 우선순위 낮음.

[ 063 ] 수분부족분(g/㎥) | xhumlack | g/m³
  판정: ⚠️ 014와 동일 API_ID — 중복 항목
  의견: API_ID가 014(수분부족분)와 동일(xhumlack).
        양액기 시스템이 복합환경제어기의 동일 센서 데이터를 재사용하는 구조로 추정.
        mock CSV에서도 컬럼이 동일. 014로 통합 처리, 이 항목은 중복 제거 권장.

[ 064 ] 배관EC(dS/m) | pi_ec | dS/m
  판정: ✅ 꺾은선 위젯 (순환 재배 시스템에서 중요)
  의견: 배관 내(배액/순환액) EC.
        순환재배(재사용) 시스템에서 배액 EC 관리에 핵심.
        급액EC(051), 배지EC(059)와 함께 3종 멀티라인으로 구성 시
        급액 → 배지 흡수 → 배액 흐름의 EC 변화 전체를 시각화 가능.
        비순환(버림재배) 시스템에서는 우선순위 낮음.

[ 065 ] 관수일사량 | g1 | -
  판정: ✅ 꺾은선 위젯 (관수 이벤트 분석용)
  의견: 관수 트리거 시점의 일사량 기준값.
        각 관수 이벤트가 어떤 일사 조건에서 발생했는지 추적.
        단위 미확정("-"). 관수 이벤트 타임라인과 함께 표시 권장.
        일반 대시보드보다 관수 분석 전용 위젯에 더 적합.

[ 066 ] 관수 시 30분간 누적일사량 | j30m | J/cm²
  판정: ✅ 꺾은선 위젯 (관수 타이밍 최적화 분석)
  의견: 관수 전후 30분 누적일사량.
        관수 트리거 임계값 설정 최적화에 활용.
        일반 실시간 모니터링보다 사후 분석/리포트 용도에 더 적합.

[ 067 ] 전일 누적일사량 | jday | J/cm²
  판정: ✅ 현재값 위젯 (일 단위 참조값)
  의견: 전날 하루 총 누적일사량. 오늘 관수 목표 설정의 기준값.
        변동이 없는 일 단위 고정값이므로 꺾은선보다 현재값 표시.
        최근 7일 추이를 작은 바 차트로 함께 표시 시 날씨 패턴 파악 용이.
