※ 노동 데이터 구조 (4개 레이어)
   301~313 : 작업자 마스터 (인적 정보)
   401~409 : 작업 마스터 (업무 정의)
   501~517 : 작업 지시·이행 (일별 운영 데이터)
   601~608 : AI 비전 평가 (작업 품질 결과)

※ 노동 데이터의 특성
   - 마스터(300·400번대)는 정적 참조 데이터 → 설정/관리 화면 표시
   - 운영(500번대)은 일별 갱신 → 작업 현황 대시보드
   - 평가(600번대)는 작업 완료 후 확정 → 성과 분석 뷰
   - '실시간 수치 추이' 위젯보다 '상태·현황 패널' 및 '목록 위젯'이 적합

[ 작업자 마스터 (301~313) ]
================================================================================

[ 301 ] 작업자 ID | user_id | 문자열
  판정: ❌ 위젯 불필요 — 작업자 목록 테이블 기본키
  의견: "W-001" 형태의 식별자. 수치 위젯 불가.
        302~313의 작업자 카드/테이블의 Primary Key로 활용.

[ 302 ] 역할 | role | 문자열 (Manager / Worker)
  판정: ⚠️ 필터 기준값 — 독립 위젯 불필요
  의견: Manager / Worker 구분. 위젯 단독 표시 의미 없음.
        작업자 목록 화면의 필터 탭(전체 / 관리자 / 작업자) 기준값으로 활용.

[ 303 ] 이름 | name | 문자열
  판정: ❌ 위젯 불필요 — 작업자 카드/목록 표시 항목
  의견: 작업자 카드 UI의 표시명. 개인정보 처리 주의.

[ 304 ] 프로필 이미지 URL | profile_img_url | URL
  판정: ⚠️ 작업자 카드 UI 아바타 이미지 — 수치 위젯 불가
  의견: 작업자 카드에 아바타 이미지 표시용 URL.
        미지원 시 이니셜 아바타로 대체 설계 권장.

[ 305 ] 연락처 | contact | 문자열
  판정: ❌ 위젯 불필요 — 작업자 카드 상세 정보 항목
  의견: 개인정보. 권한에 따른 접근 제한 설계 필요.

[ 306 ] 선호 언어 | preferred_language | 문자열 (KR / VN / TH / PH)
  판정: ✅ 작업 지시 언어 설정 활용 (위젯보다 기능 연동)
  의견: 외국인 근로자 지시서(SOP) 자동 번역 언어 선택 기준.
        독립 위젯 불필요. 작업 지시(407 sop_text_kr) 자동 번역 트리거로 활용.
        ※ 다국어 SOP 기능 구현 시 핵심 메타데이터.

[ 307 ] 국적 | nationality | 문자열
  판정: ❌ 위젯 불필요 — 작업자 카드 항목
  의견: 306(선호언어)와 연계. 인력 구성 통계(국적별 비율) 차트에 활용 가능.
        예: 원형 차트로 KR 30% / VN 40% / TH 20% / PH 10% 표시.

[ 308 ] 비자 상태 | visa_status | 문자열 (내국인 / E-8)
  판정: ⚠️ 상태 배지 (만료 경보 연동 권장)
  의견: E-8 등 취업비자 상태. 비자 만료일 관리와 연동 필요.
        만료 30일 이내 작업자에 주황 배지, 7일 이내 빨간 배지 경보 권장.
        310(계약 기간)과 함께 '인력 만료 관리' 기능으로 확장 가능.

[ 309 ] 고용 형태 | employment_type | 문자열 (정규직 / 계절)
  판정: ❌ 위젯 불필요 — 작업자 카드 항목
  의견: 인력 구성 통계에서 활용 가능 (정규직 vs 계절 비율).

[ 310 ] 계약 기간 | contract_period | 문자열 (날짜 범위)
  판정: ⚠️ D-day 계약 만료 카운트다운 (308과 연계)
  의견: 계약 만료 임박 인력 관리에 활용.
        308(비자 상태)과 함께 '인력 계약 만료 알림' 기능으로 구현.
        단순 날짜 표시보다 "계약 만료 D-30" 형태 알림 카드 권장.

[ 311 ] 소속 기관/에이전시 | agency_name | 문자열
  판정: ❌ 위젯 불필요 — 작업자 카드 항목
  의견: 직고용 / 외국인 파견 에이전시 구분. 인력 공급처 관리 목록에서 활용.

[ 312 ] 가능 작업 | capable_tasks | 문자열 (쉼표 구분)
  판정: ✅ 작업 배정 기능 연동 (태그 형태 표시)
  의견: "수확, 적엽, 곁순제거" 등 작업 가능 목록.
        작업 지시(501~517) 배정 시 적격자 자동 필터링 기준값.
        작업자 카드에 태그 형태로 표시 권장.

[ 313 ] 배정 가능 구역 | allowed_zones | 문자열 (쉼표 구분)
  판정: ✅ 구역 배정 자동화 연동 (태그 형태 표시)
  의견: "1구역, 2구역" 등 작업 허용 구역.
        505(작업 지시 구역)와 매칭하여 적격 작업자 자동 필터링.
        구역 지도 UI와 연동 시 시각적 배정 가능.

[ 401 ] 작업 코드 | task_code | 문자열 (TSK-01~04)
  판정: ❌ 위젯 불필요 — 작업 마스터 테이블 기본키
  의견: 작업 정의 식별자. 402~409의 기준 키.

[ 402 ] 작업명 | task_name | 문자열
  판정: ⚠️ 작업 마스터 카드/목록 표시 항목
  의견: "수확", "적엽", "곁순제거", "방제" 등.
        작업 지시(501) 생성 시 선택 드롭다운 항목.
        독립 위젯 불필요.

[ 403 ] 난이도 | difficulty_level | 문자열 (상/중/하)
  판정: ⚠️ 배정 필터 기준 — 독립 위젯 불필요
  의견: 작업 난이도. 312(가능 작업)의 숙련도 수준과 매칭하여 적격자 배정에 활용.
        작업 마스터 카드에 배지 형태 표시.

[ 404 ] 필요 인원 | required_headcount | 명
  판정: ✅ 현재 배정 인원 대비 부족 경보 위젯
  의견: 작업별 최소 필요 인원수.
        당일 실제 배정 인원(504 집계)과 비교하여 인원 부족 경보 표시 가능.
        예: 수확 작업 최소 5명 필요 → 현재 3명 배정 → 빨간 경보.

[ 405 ] 표준 작업 시간 | standard_time | 분
  판정: ⚠️ 작업 소요 시간 비교 참조값 — 독립 위젯 불필요
  의견: 작업별 표준 소요 시간.
        512~513(실제 시작/종료 시간)과 비교하여 작업 효율성 분석에 활용.
        실제/표준 시간 비율 = 숙련도 지표로 활용 가능.

[ 406 ] 단위 | unit_type | 문자열 (kg / bed / row / zone)
  판정: ❌ 위젯 불필요 — 표시용 단위 레이블
  의견: 작업량 표시 단위. 위젯 라벨로만 활용.

[ 407 ] SOP (표준작업절차) | sop_text_kr | 문자열
  판정: ⚠️ SOP 팝업/카드 표시 — 수치 위젯 불가
  의견: 작업 지시 시 작업자에게 전달되는 절차 안내.
        306(선호 언어) 기반 자동 번역 연동 시 다국어 SOP 제공 가능.
        작업 지시 카드 내 "SOP 보기" 버튼으로 팝업 표시 권장.

[ 408 ] 가이드 이미지 URL | guide_image_urls | URL
  판정: ⚠️ SOP 이미지 첨부 — 수치 위젯 불가
  의견: 407(SOP 텍스트)와 함께 작업 가이드 팝업에 표시.
        외국인 근로자 대상 비언어적 작업 지시에 중요.

[ 409 ] 평가 기준 | eval_criteria | 문자열
  판정: ⚠️ 평가 카드 참조 표시 — 수치 위젯 불가
  의견: "상처 없는 과실 비율", "줄기 손상 여부" 등 평가 항목.
        601~608(AI 평가 결과)와 연계하여 평가 기준 설명 표시.

[ 501 ] 작업 지시 번호 | order_id | 문자열

[ 502 ] 작업 코드 (매핑) | task_code_mapped | 문자열
  판정: ❌ 위젯 불필요 — 401 작업 마스터 연결 키
  의견: 401(task_code) 참조 외래키.

[ 503 ] 지시자 ID | assigner_id | 문자열
  판정: ❌ 위젯 불필요 — 작업 지시 이력 항목
  의견: 작업 지시한 관리자 ID. 이력 테이블 항목.

[ 504 ] 배정 작업자 ID | assignee_id | 문자열
  판정: ❌ 위젯 불필요 — 작업 배정 이력 항목
  의견: 작업 배정된 작업자 ID. 301(작업자 마스터)와 연결 키.

[ 505 ] 대상 구역 | target_zone_id | 문자열
  판정: ⚠️ 구역별 작업 현황 위젯 연동 기준값
  의견: "Zone-1" 등 작업 실시 구역.
        구역별 금일 작업 건수/완료율 집계 시 기준값으로 활용.
        구역 지도 UI에서 구역별 색상 표시(작업 완료율 기반) 가능.

[ 506 ] 우선순위 | priority | 문자열 (긴급/높음/보통)
  판정: ✅ 작업 목록 정렬 + 긴급 알림 배지
  의견: 긴급/높음/보통 분류.
        작업 목록 정렬 기준. 긴급 건 발생 시 상단 고정 + 빨간 배지 알림.
        당일 긴급 작업 수를 TopBanner 또는 알림 인디케이터로 표시 권장.

[ 507 ] 목표 작업량 | target_amount | 단위(406 기준)
  판정: ✅ 달성률 게이지 위젯 (514와 쌍)
  의견: 작업 지시 목표량.
        514(완료량)과 비교하여 달성률 게이지로 표시.
        예: 수확 목표 120kg → 실제 완료 113.5kg (94.6%)
        작업 목록 카드에 달성률 진행바 표시 권장.

[ 508 ] 계획 시작 시각 | planned_datetime | 날짜시각
  판정: ⚠️ 타임라인 위젯 기준값 — 독립 위젯 불필요
  의견: 작업 계획 시간. 512(실제 시작)과 비교하여 시작 지연 여부 파악.
        타임라인 간트 차트 형태 위젯에서 계획 vs 실제 비교 시각화 가능.

[ 509 ] 작업 메모 | order_memo | 문자열
  판정: ❌ 위젯 불필요 — 작업 지시 카드 내 텍스트 표시
  의견: 작업자 전달 메모. 작업 카드 하단 텍스트 영역으로 표시.

[ 510 ] 작업 상태 | status | 문자열 (완료/이슈발생)
  판정: ✅ 작업 현황 상태 패널 (핵심 위젯)
  의견: 당일 전체 작업 건수 중 완료 / 이슈발생 / 진행중 현황.
        금일 작업 현황을 집계하여:
          완료 N건 / 이슈발생 N건 / 진행중 N건
        형태의 현황 패널 + 도넛 차트로 표시 강력 권장.
        이슈 발생 건수가 0 이상이면 즉시 알림 배지 활성화.

[ 511 ] 수신 확인 | read_receipt | boolean
  판정: ⚠️ 작업 지시 목록에서 미확인 배지로 표시
  의견: 작업자의 지시서 확인 여부.
        미확인(false) 건수를 집계하여 알림 인디케이터로 표시.
        예: 작업자 3명 미확인 → 관리자 알림.

[ 512 ] 실제 시작 시각 | actual_start_time | 날짜시각
  판정: ⚠️ 타임라인 위젯 (508 계획과 비교)
  의견: 508(계획 시작)과 비교하여 지연 시간 자동 계산.
        예: 계획 09:00 → 실제 09:07 → 7분 지연.

[ 513 ] 실제 종료 시각 | actual_end_time | 날짜시각
  판정: ⚠️ 타임라인 위젯 (405 표준 시간과 비교)
  의견: 512(실제 시작)~513(실제 종료) 차이 = 실제 소요 시간.
        405(표준 시간) 대비 효율 지표 계산에 활용.
        작업 완료 후 자동 기록되므로 실시간 위젯보다 완료 후 분석 뷰에 적합.

[ 514 ] 완료 작업량 | completed_amount | 단위(406 기준)
  판정: ✅ 달성률 게이지 위젯 (507 목표량과 쌍)
  의견: 507(목표량) 대비 실제 완료량. 달성률 자동 계산.
        100% 미만이면 미완성 → 지연 위험 경보와 연계 가능.
        작업 카드의 핵심 성과 지표.

[ 515 ] 이슈 유형 | issue_type | 문자열 (정상/장비이상/시간부족)
  판정: ✅ 이슈 현황 패널 + 510(상태)과 연계
  의견: 이슈 발생 시 세부 분류.
        당일 이슈 유형별 집계(장비이상 N건 / 시간부족 N건) 차트 표시.
        반복 이슈 유형은 시설 관리 또는 인력 관리 개선 신호.

[ 516 ] 미완성 사유 | incomplete_reason | 문자열
  판정: ❌ 위젯 불필요 — 이슈 로그 텍스트 항목
  의견: "작업 속도 저하로 인한 시간 부족" 등 텍스트.
        이슈 상세 팝업에서 텍스트 표시.

[ 517 ] 작업자 피드백 | worker_feedback | 문자열
  판정: ❌ 위젯 불필요 — 작업 완료 리포트 텍스트
  의견: "가위 날이 무뎌 교체 필요함" 등 자유 형식 텍스트.
        장비 유지관리 알림과 연계 가능.
        예: '장비이상' 피드백 N건 이상 시 → 시설팀 알림 자동 발송.

  판정: ❌ 위젯 불필요 — AI 평가 이력 식별자
  의견: "RV-20260313-123" 형태의 로봇 스캔 ID. 이력 추적 기준 키.

[ 603 ] 로봇 스캔 시각 | robot_scan_time | 날짜시각

  의견: AI 평가 수행 시각. 이력 로그 항목.

[ 604 ] AI 평가 점수 | ai_eval_score | 점 (0~100)
  판정: ✅ 현재값 KPI 카드 + 일별 추이 바 차트 (핵심 성과 지표)
  의견: 수치형 점수. 작업자별 당일 AI 평가 점수.
        전체 작업자 평균 점수를 TopBanner KPI 후보로 표시.
        606(최종 등급)과 연계: 90점 이상=A, 80점 이상=B, 미만=C.
        작업자별 점수 분포를 바 차트 또는 히트맵으로 표시 강력 권장.
        일별 추이 꺾은선으로 AI 평가 점수 트렌드 파악 가능.

[ 605 ] AI 감지 이슈 | ai_detected_issues | 문자열
  판정: ⚠️ 이슈 빈도 집계 차트 (텍스트 분류 필요)
  의견: "미수확 과실 잔존", "줄기 꺾임", "특이사항 없음" 등.
        개별 텍스트는 위젯 불가. 분류 집계 시 의미 있음.
        이슈 유형별 발생 빈도 바 차트로 시각화 가능.
        예: 이번 주 미수확 잔존 8건 / 줄기 손상 3건.

  의견: A(우수) / B(보통) / C(미흡) 3단계.

[ 607 ] 인센티브 포인트 | incentive_points | 점
  판정: ✅ 작업자 포인트 현황 위젯 (개인 대시보드 연동)
  의견: A등급 달성 시 604점수 × 10 포인트.
        작업자별 누적 인센티브 포인트 랭킹 차트.
        관리자 뷰: 팀 전체 인센티브 집계.
        작업자 개인 뷰: 본인 누적 포인트 + 랭킹.
        성과 보상 동기부여 기능으로 확장 가능.

[ 608 ] 평가자 메모 | evaluator_memo | 문자열
  판정: ❌ 위젯 불필요 — 평가 카드 텍스트 항목
  의견: "우수함", "재교육 필요" 등 관리자 코멘트.

        C등급 작업자에 대한 재교육 일정 연계 기능으로 확장 가능.
