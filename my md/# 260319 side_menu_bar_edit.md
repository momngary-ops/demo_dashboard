# 대시보드 플랫폼 사이드 메뉴 개발 프롬프트

@claude.code, 당신은 이 프로젝트의 프론트엔드 개발 AI Agent로 참여합니다.
아래 제시된 요구사항을 바탕으로 대시보드의 사이드 메뉴(LNB) 컴포넌트를 개발해 주세요.

---

## 1. 메뉴 계층 구조

### [Dashboard] ← 기존 네비게이션 명 유지 / 클릭 가능 / 랜딩 페이지
> [Dashboard] 헤더 클릭 = 대시보드 홈과 동일 화면. 랜딩 페이지 정책 적용.

- **대시보드 홈** — 현재 구현된 대시보드 페이지
- **My Dashboard** — 개인 커스텀 대시보드 (모든 사용자 권한 동일)
  - 레이아웃은 [Dashboard]와 동일
  - KPI 카드 및 하위 위젯 항목 모두 사용자가 자유롭게 설정 가능
  - 구역(Zone) 랜딩 정책 (우선순위 순):
    1. 사용자가 설정한 랜딩 구역 → 해당 구역 (최우선)
    2. 설정 없음 + 캐시된 마지막 구역 있음 → 마지막 본 구역
    3. 설정·캐시 모두 없음 → 1구역 기본 랜딩

### [AI Agent] ← 클릭 불가 / 클릭 시 하위 메뉴 아코디언 펼침·접힘만 동작
- AI컨설팅리포트
- AI매뉴얼 Agent
- 환경진단 Agent
- 생육진단 Agent
- AI 농장운영 분석

### [데이터입력] ← 클릭 불가 / 클릭 시 하위 메뉴 아코디언 펼침·접힘만 동작
- 환경데이터 입력
- 생육데이터 입력
- 경영데이터 입력

### [데이터설정] ← 클릭 불가 / 클릭 시 하위 메뉴 아코디언 펼침·접힘만 동작
- 데이터 항목 수정 및 변경
- 대시보드 초기화

> ※ 현재 구현 범위: 모든 하위 메뉴 클릭 시 "준비 중" 페이지 표시 (추후 개별 페이지 연결 예정)

## 2. 확정된 정책 및 동작 정의

| 항목 | 내용 |
|------|------|
| 기술 스택 | React 19 + Vite, lucide-react 아이콘 |
| 라우팅 방식 | URL 없음 — `onNavigate(id)` state 기반 전환 유지 |
| 사이드바 접힘 | 56px ↔ 200px, CSS transition (기존 구현 유지) |
| 활성 상태 | `activePage === id` prop 비교 (기존 방식 유지) |
| [Dashboard] 헤더 클릭 | 대시보드 홈 페이지로 이동 (랜딩 페이지 정책) |
| [AI Agent] 등 헤더 클릭 | 페이지 이동 없음, 아코디언 펼침·접힘만 동작 |
| 기존 메뉴 처리 | `데이터 조회`, 영문 `Dashboard` (중복 항목) 삭제 |
| Account 변경 | Our community → Our Team, Contact Us 삭제 |

## 5. 개발 계획 (Plan)

### STEP 1 — Sidebar.jsx 메뉴 데이터 구조 재정의
- 기존 flat 배열 → 그룹 + 하위 메뉴 중첩 구조로 변경
- 그룹 타입 구분: `type: 'link'` (Dashboard) vs `type: 'accordion'` (나머지)
- Account 섹션: Our Team / Profile / Logout 으로 교체

### STEP 2 — 아코디언 상태 관리 추가
- `openGroups` state (Set) 추가: 어떤 그룹이 펼쳐져 있는지 관리
- 그룹 헤더 클릭 시 toggle 동작 (AI Agent, 데이터입력, 데이터설정)
- 사이드바 접힌 상태에서는 아코디언 무효화 (아이콘만 표시)

### STEP 3 — 서브메뉴 렌더링 및 스타일
- 서브메뉴 항목 들여쓰기 + 작은 점(·) 또는 선 구분자
- 펼침/접힘 CSS transition 애니메이션 (height: 0 → auto)
- 활성 서브메뉴 항목 하이라이트

### STEP 4 — 준비 중 페이지 처리
- `onNavigate` 호출 시 구현되지 않은 페이지 ID → App.jsx에서 "준비 중" fallback 렌더링

### STEP 5 — 대시보드 초기화 확인 모달
- `[데이터설정] > 대시보드 초기화` 클릭 시 인라인 confirm 모달 표시
- 확인 시 초기화 로직 실행, 취소 시 모달 닫기
