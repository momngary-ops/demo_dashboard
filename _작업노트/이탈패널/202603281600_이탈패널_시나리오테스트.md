# 이탈 패널 — 개별 닫기 기능 시나리오 테스트

**날짜**: 2026-03-28
**대상 기능**: DeviationPanel 개별 카드 닫기 (✕ 버튼) + 접힘 요약 표시
**수정 파일**: `DeviationPanel.jsx`, `DeviationPanel.css`, `DashboardPage.jsx`

---

## 수정 배경

기존 코드는 닫기 상태(`hiddenIds`)를 DeviationPanel 내부에서 관리했으나,
KPI 폴링으로 `slots` 레퍼런스가 교체될 때마다 `useEffect([slots])`가 실행되어
닫은 카드가 다시 나타나는 레이스 컨디션이 존재했음.

**해결 방향**: 닫기 상태를 DashboardPage의 `dismissedDeviationIds`(Set)로 끌어올리고,
DeviationPanel은 `onDismiss(id)` 콜백으로 상위에 위임.

---

## 시나리오 테스트 결과

### S1 · 센서 1개 이탈 중 ✕ 눌렀을 때
**기대**: 카드가 사라지고 패널도 함께 닫힌다.
**결과**: ✅ 통과
- `onDismiss('A')` → `dismissedDeviationIds = {A}`
- `visibleDeviatedSlots = []` → 패널 언마운트
- cleanup effect: 센서 A는 여전히 이탈 중이므로 dismissedIds 유지

---

### S2 · 카드를 닫은 뒤 해당 센서가 정상으로 돌아왔을 때
**기대**: "닫힘" 기록이 자동으로 제거되어 깨끗하게 초기화된다.
**결과**: ✅ 통과
- 센서 복귀 → `deviatedSlots = []`
- cleanup effect: `currentIds = {}` → A가 currentIds에 없음 → `dismissedIds = {}` 자동 정리
- 불필요한 기록이 남지 않음

---

### S3 · 닫았다가 정상 복귀 후 다시 이탈했을 때
**기대**: 카드가 다시 나타나고 패널이 자동으로 펼쳐진다.
**결과**: ✅ 통과
- S2에서 `dismissedIds = {}` 정리됨
- 재이탈 → `deviatedSlots = [A]`, `dismissedIds = {}` → `visibleDeviatedSlots = [A]`
- `prevDeviatedIdsRef`가 비어있었으므로 `hasNew = true` → `expandTrigger` 증가 → 자동 펼침

---

### S4 · "모두 닫기" 버튼으로 한 번에 전체 닫을 때
**기대**: 카드가 여러 개여도 동시에 한 번에 처리된다.
**결과**: ✅ 통과
- `handleDismissAll()` → `forEach`로 `onDismiss` 3회 호출
- React 18 자동 배칭 + 함수형 updater 체이닝:
  `{} → {A} → {A,B} → {A,B,C}` 순서로 올바르게 누적
- 단일 리렌더로 처리

---

### S5 · 일부 카드를 닫아둔 상태에서 새로운 센서가 이탈할 때
**기대**: 닫아둔 카드는 유지되고 새 카드만 추가로 표시된다.
**결과**: ✅ 통과
- `dismissedIds = {A}`, 신규 B 이탈 → `deviatedSlots = [A, B]`
- cleanup effect: A가 여전히 currentIds에 있음 → dismissedIds 유지 (`{A}` 그대로)
- `visibleDeviatedSlots = [B]` → B만 표시, A 카드는 노출 안 됨
- B는 신규이므로 `expandTrigger` 증가 → 자동 펼침

---

### S6 · 패널을 접은 채로 닫았다가 센서가 다시 이탈할 때
**기대**: 패널이 접히지 않고 펼쳐진 상태로 다시 등장한다.
**결과**: ✅ 통과
- 패널 접힘(`collapsed = true`) → 모두 닫음 → 패널 언마운트
- 재이탈 → 패널 재마운트 → `useState(false)` 초기값으로 펼쳐진 상태
- `expandTrigger` effect → `setCollapsed(false)` (이미 false, no-op)

## 발견 및 수정한 버그

### BUG-A · CSS `:has()` 선택자 — 브라우저 호환성 문제
**내용**: 접힌 상태의 여백 처리를 최신 CSS `:has()` 문법으로 작성해,
Firefox 120 이하 등 일부 브라우저에서 레이아웃이 깨질 수 있었음.

**수정**: `dev-panel--collapsed` 클래스를 JSX에서 직접 붙이는 방식으로 교체.
모든 브라우저에서 동일하게 동작함.

```css
/* Before */
.dev-panel:has(.dev-panel__summary) { padding-bottom: 10px; }

/* After */
.dev-panel--collapsed { padding-bottom: 10px; }
```

---

### BUG-B · `extraCount` 음수 오염
**내용**: 이탈 센서가 3개 미만일 때 `slots.length - MAX_CHIPS` 계산 결과가
음수(-1, -2)가 되어 변수가 오염될 수 있었음.
화면 표시에는 `extraCount > 0` 조건이 있어 렌더 영향은 없었으나
변수 의미 자체가 틀렸음.

**수정**: `Math.max(0, slots.length - MAX_CHIPS)` 로 음수 방지.

```js
// Before
const extraCount = slots.length - MAX_CHIPS

// After
const extraCount = Math.max(0, slots.length - MAX_CHIPS)
```

## 최종 데이터 흐름 (수정 후)

```
alertSlots (KPI 폴링)
  └─ deviatedSlots (OUT_OF_RANGE 필터)
        ├─ [cleanup effect] 복귀 센서 → dismissedDeviationIds 자동 정리
        ├─ [expand effect]  신규 이탈 → expandTrigger 증가
        └─ visibleDeviatedSlots (dismissedIds 제외)
              └─ DeviationPanel(slots, onDismiss)
                    ├─ ✕ 클릭 → onDismiss(id) → dismissedDeviationIds += id
                    └─ 모두 닫기 → forEach onDismiss → 배칭 처리
```
