## 의사결정 확정

### 1. FLAP_THRESHOLD — GuidelineSettings 알림 섹션에 연동

- `useAlertNotifier.js`의 하드코딩 `FLAP_THRESHOLD = 3` 제거
- `useGuideline()` 컨텍스트에서 `alertConfig.flapThreshold`를 읽어 동적 적용
- fallback: `glAlertConfig?.flapThreshold ?? 3`
- **저장 위치**: 서버 `alert_config` (GuidelineSettings → `/api/guidelines` → 저장)
- **UI 위치**: GuidelineSettingsPage 알림 섹션 하단, **공통 탭에서만** 표시
  - 범위: 2~10회, step 1
  - 레이블: `FLAPPING 감지 — 딜레이 내 [N]회 이상 반복 이탈 시 알림`

### 2. RECOVERED 인앱 알림 — Teams만 전송 (인앱 억제)

- `useAlertNotifier.js` RECOVERED 블록에서 `addNotification()` 호출 제거
- `sendTeamsRecovery()` 는 그대로 유지
- 벨 패널 / 토스트에 RECOVERED 항목 표시 안 함
- NotificationPanel / NotificationToast의 RECOVERED 메타 코드는 잔류 (향후 재활성화 가능)

```js
// FLAP_THRESHOLD → 동적 읽기

if (wasSent(slot.id, prev) && isInCooldown(slot.id, prev, cooldownMin)) {

    sendTeamsRecovery(cfg.webhookUrl, { ...slot, prevStatus: prev })
      .catch(err => console.warn('[Alert] Teams RECOVERED 알림 실패:', err.message))
  }
}

// useEffect deps
}, [slots, addNotification, glAlertConfig])
```

## 알림 정책 최종 확정

```
이탈 감지 → 딜레이 타이머 시작
  ├─ 딜레이 내 복귀 → flapCount++
  │     flapCount < flapThreshold  → 타이머 리셋, 알림 없음
  │     flapCount >= flapThreshold → FLAPPING 발송 (인앱 + Teams), flapCount 리셋
  └─ 딜레이 경과 유지
        쿨다운 중    → 알림 없음
        쿨다운 아님  → 경고 발송 (인앱 + Teams), cd키 기록, flapCount 리셋

정상 복귀
  ├─ cd키 있음 + 쿨다운 내 → Teams RECOVERED만 발송, cd키 삭제
  └─ cd키 없음 or 쿨다운 아님 → 알림 없음, cd키 삭제
```
