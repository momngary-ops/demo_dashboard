# Teams 알림 정책 및 Adaptive Card 고도화 — 확정

**날짜**: 2026-03-27
**상태**: 정책 확정, 구현 대기

---

## 1. 알림 발송 정책 (확정)

```
이탈 감지
  │
  ├─ [판정 보류] → 조용히 무시
  │     - 항목별 알림 disabled
  │     - 가이드라인 미로드 (서버 연결 실패)
  │     - 가이드라인 값 0/null, min>max, CO₂ 참고치 0
  │
  └─ 딜레이 타이머 시작 (alertConfig.delay_min)
       │
       ├─ 딜레이 내 복귀
       │     flapCount++
       │     ├─ flapCount 1~2 → 타이머 리셋, 알림 없음 (노이즈 간주)
       │     └─ flapCount 3+  → [FLAPPING] 알림 발송 → flapCount 리셋
       │
       └─ 딜레이 경과까지 이탈 유지
             ├─ 쿨다운 중 (cooldownMin 이내 동일 이전 발송) → 알림 없음
             └─ 쿨다운 아님
                  → [경고] 알림 발송
                  → cd키 기록: alert:cd:{id}:{status} = Date.now()
                  → flapCount 리셋

  정상 복귀 시
       ├─ cd키 있음 + isInCooldown = TRUE
       │    → [RECOVERED] 알림 발송
       │    → cd키 삭제
       └─ cd키 없음 or isInCooldown = FALSE
            → 알림 없음, cd키 삭제만
```

---

## 2. 알림 타입 정의

| 타입 | 트리거 | 성격 | 신규 여부 |
|---|---|---|---|
| `OUT_OF_RANGE` | 딜레이 경과 + 쿨다운 아님 | 경고 | 기존 |
| `FLAPPING` | 딜레이 내 복귀 3회+ | 반복 이탈 경고 | **신규** |
| `STALE_CRIT` | 데이터 수신 중단 | 경고 | 기존 |
| `SENSOR_FAULT` | 센서 오류값 감지 | 경고 | 기존 |
| `SENSOR_LOST` | 센서 연결 끊김 | 경고 | 기존 |
| `RECOVERED` | 쿨다운 내 정상 복귀 | 복구 | **신규** |

## 3. Adaptive Card 레이아웃 (MessageCard 대체)

### 경고 카드 (OUT_OF_RANGE / FLAPPING)
```
[주황 스트립]
⚠️ 임계치 이탈                그린스케이프 대동팜
─────────────────────────────────────────────
🌡️ 내부 온도  ·  1구역
─────────────────────────────────────────────
현재값              이탈 범위
30.2°C              +5.2°C
                    상한 25.0°C
─────────────────────────────────────────────
이탈 확정           발생시각
3분 이상 지속       2026.3.27  15:30
```
- FLAPPING: 제목 → `⚠️ 반복 이탈 감지`, 이탈 확정 → `딜레이 내 N회 반복`

### 복구 카드 (RECOVERED)
```
[초록 스트립]
✅ 정상 복귀                  그린스케이프 대동팜
─────────────────────────────────────────────
🌡️ 내부 온도  ·  1구역
─────────────────────────────────────────────
현재값              이전 상태
24.8°C              임계치 이탈
─────────────────────────────────────────────
복구 시각: 2026.3.27  16:15
```

### 장애 카드 (STALE_CRIT / SENSOR_FAULT / SENSOR_LOST)
```
[빨강 스트립]
🔴 데이터 수신 중단            그린스케이프 대동팜
─────────────────────────────────────────────
🌡️ 내부 온도  ·  1구역
─────────────────────────────────────────────
마지막 수신값       발생시각
30.2°C              2026.3.27  15:30
```

## 4. Adaptive Card 색상 정책

| 타입 | 스트립 색상 | Adaptive Card 키워드 |
|---|---|---|
| OUT_OF_RANGE, FLAPPING | 주황 `#F97316` | `warning` |
| STALE_CRIT, SENSOR_FAULT, SENSOR_LOST | 빨강 `#EF4444` | `attention` |
| RECOVERED | 초록 `#4ADE80` | `good` |

## 6. 이탈 지속시간 표시 정책

- **A안 채택 (간단)**: `alertDelayMin` 기반으로 `"N분 이상 지속"` 표시
- B안 (정확한 경과시간, deviationStats 연동)은 추후 개선으로 보류

## 8. 미결 사항

- flapCount 임계값 3회 — 추후 가이드라인 설정 UI에서 조정 가능하도록 고려
- RECOVERED 인앱 알림 표시 여부 (노이즈 우려 시 Teams만 발송으로 제한 가능)
