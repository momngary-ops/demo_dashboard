# 생육 대시보드 — 개요 및 통합 계획

> 작성일: 2026-03-17

---

## 1. 목적

토마토 생육 데이터를 **로봇 수집(API) + 수기 입력(Manual)** 방식으로 입력·저장하는 전용 페이지.
현재는 Flask 단독 서버(app.py)로 프로토타입 구현, 이후 메인 대시보드에 통합 예정.

---

## 3. 메인 대시보드 통합 계획

### 통합 위치
```
src/
├── pages/
│   ├── DashboardPage.jsx       ← 기존
│   └── DataInputPage.jsx       ← 신규 (app.py → 변환)
└── App.jsx                     ← PAGES 맵에 'data-input' 추가
```

### App.jsx 변경 사항
```jsx
// 기존
const PAGES = {
  dashboard: <DashboardPage />,
}

// 변경 후
import DataInputPage from './pages/DataInputPage'

const PAGES = {
  dashboard:  <DashboardPage />,
  'data-input': <DataInputPage />,
}
```

### Sidebar.jsx — 이미 준비됨
```js
// NAV_ITEMS에 이미 포함되어 있음
{ id: 'data-input', label: '데이터 입력', icon: ClipboardEdit },
```

---

## 4. 통합 시 제거할 항목

app.py → DataInputPage.jsx 변환 시 아래 항목은 이미 App.jsx가 제공하므로 제거:
- `<Sidebar />` 컴포넌트 (App.jsx에서 렌더링)
- `<Header />` 컴포넌트 (App.jsx에서 렌더링)
- CSS 변수 정의 (index.css에 이미 선언)
- Flask 서버 코드 전체

`DataInputPage.jsx`에는 **`<div class="page"> ... </div>`** 내부만 남김.
