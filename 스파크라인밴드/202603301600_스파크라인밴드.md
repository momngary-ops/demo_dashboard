## 변경 1: 스파크라인 경계 끊김 수정 (`Widget.jsx`)

### 문제
- 정상↔이탈 색상 전환 구간에서 선이 끊어진 것처럼 보임
- 원인: 세그먼트마다 독립 `buildPath()` → 끝점에 `strokeLinecap="round"` 캡이 겹쳐 틈 발생

### 해결: clipPath 마스킹 방식으로 전환

| 항목 | 이전 | 이후 |
|------|------|------|
| 경로 생성 | 세그먼트별 독립 `buildPath(si, ei)` | 전체 `fullD` 1개 생성 |
| 색상 분리 | 각 세그먼트 별도 `<path>` 요소 | 동일 `fullD`를 `clipPath`로 구간 마스킹 |
| 경계 처리 | 두 path가 만나지 않아 틈 발생 | 인접 포인트 X 중간점으로 clipRect 분할 → C1 연속 |
| SVG id | `spk-grad` 고정 (다중 위젯 충돌) | `useRef` 기반 랜덤 `uid` — 인스턴스별 고유 |

### 핵심 코드 패턴
```jsx
// defs 안에 세그먼트별 clipPath 정의
<clipPath id={`${uid}-${si}`}>
  <rect x={leftX} y={0} width={rightX - leftX} height={H} />
</clipPath>

// 같은 fullD path를 각 세그먼트 색상으로 렌더링, clipPath로 마스킹
<path d={fullD} stroke={seg.inRange ? COLOR_IN : COLOR_OUT}
      clipPath={`url(#${uid}-${si})`} ... />
```

## 변경 2: 평균 온도 위젯 API 구현 (`mock_data_server.py`)

### 문제
- `AvgTempWidget`이 `/api/zone/{id}/temp-weekly-avg` 호출 → 미구현 → `weekly=null` → 주간/일평균/야간 전부 `--`

### 해결: SQLite 집계 엔드포인트 추가

```python
GET /api/zone/{zone_id}/temp-weekly-avg
```

**반환 구조**:
```json
{
  "week_day_avg":   20.5,   // 최근 7일 주간(6~18시) 평균
  "week_night_avg": 15.2,   // 최근 7일 야간(18시 이후) 평균
  "week_daily_avg": 18.1,   // 최근 7일 일평균
  "days": [
    { "date": "2026-03-24", "day_high": 22.3, "daily_avg": 19.1, "night_low": 14.5 },
    ...
  ]
}
```

**데이터 소스**: `kpi_log` 테이블 `xintemp1` 필드, 최근 7일
**주간 기준**: `strftime('%H') BETWEEN 6 AND 17`
**야간 기준**: 그 외 (18시~익일 6시)
